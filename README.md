## 무결성과 불변성

RC3.9의 동결 원본 ZIP은 다시 압축하거나 조용히 덮어쓰지 않는다. `SHA256SUMS_ORIGINAL_RELEASE.txt`에 기록된 checksum을 기준으로 원본을 검증한다. 내용 변경이 필요하면 RC3.9를 수정하지 않고 새 버전과 새 태그를 만든다.

## GitHub Release / DOI 절차

1. 이 저장소를 Zenodo의 GitHub 연동에서 활성화한다.
2. Git tag `rc3.9`를 만든다.
3. GitHub Release 제목을 `Kkanttappiya RC3.9`로 공개한다.
4. 필요하면 동결 ZIP 두 개와 BagIt archive를 release asset으로 첨부한다.
5. Zenodo가 릴리스를 수집한 뒤 version DOI와 record URL을 이 README 및 `RELEASE_RECORD.json`의 후속 버전에 기록한다. 동결 payload 자체는 DOI 삽입을 위해 재작성하지 않는다.
